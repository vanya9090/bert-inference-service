# bert-inference-service
## Выбор метрик для бенчмарка
### Latency P95 / P99
#### Что именно она показывает для inference service
- Максимальное время, за которое сервис отвечает на 95/99% запросов
#### Какой целевой порог считаете приемлемым и почему
- Для текущей модели считаю, что приемлемый порог должен быть < 0.1с, так как вероятно, этот сервис должен использовать другими, например, RAG чат-ботом
#### Что будете делать, если порог нарушен
- Можно складывать нескольок запросов в батчи

### RPS
#### Что именно она показывает для inference service
- Сколько текстов векторизует сервис за одну секунду под нагрузкой
#### Какой целевой порог считаете приемлемым и почему
- Опять же зависит от того, где сервис используется, если сервис будет использоваться в RAG чат-боте, то RPS должен быть больше 50
#### Что будете делать, если порог нарушен
- Перенос модели на gpu
- Горизонтальное масштабирование 

### Model inference time
Я считаю, что метрика избыточно, можно обойтись без нее. Эта метрика скорее должна измеряться во время ресерча моделей

## Результаты профилирования

Нагрузочное тестирование проводилось с помощью утилиты Locust.
Условия тестирования: 20 одновременных пользователей, скорость подключения 5 пользователей в секунду.
Длительность: 1 минута.
```bash
uv run locust -f locustfile.py --headless -u 20 -r 5 -t 1m --host http://127.0.0.1:8000
```
